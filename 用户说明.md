## 概述

Hypoc 是一个程序测试与验证系统，主要功能包括：

| 功能 | 说明 |
|------|------|
| **输出检查** | 使用外部工具链（get_input_data + txt_compare）比对程序输出 |
| **格式检查** | 基于规则检查 C/C++ 代码风格（缩进、大括号、switch/case 等） |
| **字符集检查** | 检测源码是否为 GB2312 编码，支持一键转换 |

---

## 首次使用配置

**首次使用必须配置外部工具路径**，后续无需重复配置。

1. 点击主界面左下角「**配置工具**」按钮
2. 在弹出的对话框中选择：
   - `get_input_data.exe` — 提取测试数据输入
   - `txt_compare.exe` — 比对输出差异
3. 路径自动保存至 `config.json`

> 如果工具放在 `tools/` 目录下，程序会自动识别。

## 输出检查

使用外部工具链进行输出比对，复现真实测试环境。

### 预设模式

适用于已有预设题目数据的场景。

**操作步骤：**
1. 选择「**预设**」子Tab
2. 点击「**选择EXE**」上传用户程序（.exe）
3. 从下拉框选择对应的**问题**编号
4. （可选）根据需要调整「**比对参数**」
5. 点击「**开始测试**」

**原理：**
- 从 `data/checkdata/` 加载测试用例
- 期望输出从 `data/demoraw/` 的内置 JSON 还原
- 用户 EXE 输出与期望输出逐项比对

---

### 自定义模式

适用于自定义测试用例或无预设题目的场景。需要准备：
- **Demo 程序** — 正确实现，作为期望输出参照
- **用户程序** — 待测试的程序

**操作步骤：**
1. 选择「**自定义**」子Tab
2. 点击「**选择Demo**」上传正确实现的可执行文件
3. 点击「**选择用户EXE**」上传待测程序
4. 配置**测试数据**（三种方式）：
   - **预设**：从已有题目数据中选择
   - **文件**：上传自定义 checkdata 文件（格式见下方）
   - **在线构造**：手动添加测试用例（点击「+ 添加检查点」）
5. （可选）调整「**比对参数**」
6. 点击「**开始测试**」

**Checkdata 文件格式示例：**
```
[case-001]
输入内容第一行
输入内容第二行
[case-002]
输入内容
```

### 比对参数说明

这些参数会传递给 `txt_compare.exe` 进行比对控制：

| 参数 | 说明 | 适用场景 |
|------|------|----------|
| `--trim` | 去除空白：none/left/right/all | 输出含多余空格时 |
| `--lineskip` | 跳过前 N 行进行比较 | 输出头部有多余信息时 |
| `--lineoffset` | 行偏移量调整 | 输出行号有偏差时 |
| `--ignore_blank` | 忽略空行差异 | 空行不影响结果时 |
| `--ignore_linefeed` | 忽略换行符差异（\\r\\n vs \\n） | 跨平台比对时 |
| `--max_diff` | 允许的最大差异字符数 | 容许微小输出差异 |
| `--max_line` | 最大比较行数 | 输出很长时限制比较范围 |

## 源码检查

对 C/C++ 源代码进行格式和编码检查。

### 格式检查

基于 `Rule/rules.md` 中的规则自动检测代码风格问题：

| 错误类型 | 说明 |
|----------|------|
| `INDENT_4SPACE` | 缩进必须是 4 的倍数 |
| `NESTED_INDENT` | 嵌套块缩进不正确 |
| `MULTIPLE_STMTS` | 一行多条语句 |
| `LBRACE_SAME_LINE_STATEMENT` | 左大括号后不应有其他语句 |
| `RBRACE_LINE` | 右大括号后不应有其他语句 |
| `CONTROL_STMT_SAME_LINE` | 控制语句后的语句必须换行 |
| `CASE_INDENT` | case 必须比 switch 缩进 4 格 |
| `CASE_STMT_SAME_LINE` | case 后必须换行再写语句 |
| `CASE_STMT_INDENT` | case 下的语句必须再缩进 4 格 |
| `DO_WHILE_SINGLE_NO_BRACE` | do-while 单语句也必须加大括号 |
| `EMPTY_STMT_SAME_LINE` | 空语句的分号必须单独一行 |
| `TABLE_OUTPUT` | **严重违规** — 禁止打表输出 |

**操作步骤：**
1. 选择「**源码检查**」Tab
2. 点击「**选择源码文件**」上传 .cpp 或 .c 文件
3. 右侧自动显示格式检查结果

> **打表输出**是严重违规行为，如直接 `cout << 3.14` 或 `cout << 540` 等硬编码输出，会被标记为 `critical` 级别错误。

### 字符集检查

检测源码文件编码是否为 GB2312（简体中文环境常用编码）。

**操作步骤：**
1. 选择「**源码检查**」Tab
2. 上传源码文件后，字符集检查自动进行
3. 若检测到**非 GB2312** 编码，显示「**转换为 GB2312**」按钮
4. 点击按钮进行编码转换

## 超时设置

程序运行时的超时限制：

| 阶段 | 超时时间 |
|------|----------|
| get_input_data（提取输入数据） | 5 秒 |
| 用户程序 / Demo 程序运行 | 120 秒 |
| txt_compare（结果比对） | 120 秒 |

> 超时后会显示 `[超时]`，程序被强制终止。

## 项目结构

```
Hypoc/
├── main.py              # 主程序入口
├── config.json          # 工具路径配置（自动生成）
├── core/
│   ├── comparison.py    # 预设模式比对逻辑
│   ├── checkdata.py    # checkdata 格式解析
│   └── ext_comparison.py # 外部工具比对
├── ui/
│   ├── tabs.py          # UI 页面构建
│   └── cards.py         # 结果卡片组件
├── Rule/
│   ├── rules.md         # 格式检查规则文档
│   └── style_checker.py # C++ 风格检查器
├── char/
│   ├── check_gb2312.py      # GB2312 编码检测
│   └── convert_to_gb2312.py # GB2312 编码转换
├── tools/               # 外部工具目录（可选）
│   ├── get_input_data.exe
│   └── txt_compare.exe
└── data/
    ├── checkdata/       # 预设题目测试数据
    └── demoraw/        # 预设题目期望输出
```

## 运行与打包

**开发环境运行：**
```bash
python main.py
```

**打包为独立可执行文件：**
```bash
python build.py
```

打包后位于 `dist/main.dist/main.exe`，`data/` 目录已包含。首次使用需通过 UI 配置外部工具路径。
