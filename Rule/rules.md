1、正常顺序结构程序
要求
(1)函数内所有语句顺序缩进四格
(2)一条语句一行，左右括号单独一行
(3)适当加入注释、空行
(4) 若某行过长，可以分开数行，左侧对齐 / 缩进均可
实例:
#include <iostream>
using namespace std;

int main()
{
    int a;
    a=10;
    cout << a << endl;

    return 0;
}
反例:

using namespace std;

{   int a; // ← 不要这样
    a=10;
    cout << a << endl;

    return 0;
}

反例:
#include <iostream>

int main() { // ← 不要这样

    a=10;
    cout << a << endl;

    return 0;
}

2、含有单分支结构的程序
要求：
（1）分支内的语句顺序缩进四格
（2）左大括号两种方式（if后/另起一行）任选；右大括号单独一行（可缩进/不缩进）
（3）分支内只有一个语句的，可以不要大括号，但必须另起一行
实例:
// 单语句
if (...)
    语句1;

// 多语句（标准）
if (...) {
    语句1;
    语句2;
}

反例:

if (...)
{
    语句1;
}

// 右大括号错位
if (...) {
    语句1;
    }

// 同行无缩进
if (...) 语句1;

// 多语句同行
if (...) {语句1;语句2;}

（1）分支内的语句顺序缩进四格

6、含有switch/case的语句
要求：
(1)case比switch缩进4格，单独一行，后面不要再跟语句
(2)case下的语句比case再缩进4格
实例:
// 右大括号缩进对齐（层级清晰，可读性最强）
switch(x) {
    case 1:
        cout << a;
        break;
    case 2:
        ...;
    default:
        ...;
        break;
}
反例:
// 左大括号另起一行、右大括号不缩进
switch(x)
{
    case 1:
        cout << a;
        break;
    case 2:
        ...;
    default:
        ...;
        break;
}

// 右大括号不缩进
switch(x) {
    case 1:
        cout << a;
        break;
    case 2:
        ...;
    default:
        ...;
        break;
}

// case 后直接跟语句（违反格式要求）
case 1:cout << a; break;

switch内部的case必须严格遵循缩进层级，确保逻辑结构清晰
即使case下只有一条语句，也必须单独缩进一行，禁止与case同行
所有未列举的复杂嵌套结构，统一遵循「4格逐层缩进|原则

7、含有循环结构的程序（while和for）
要求：
（1）循环内的语句顺序缩进四格
（2）右大括号单独占一行，左大括号两种方式任选
（3）循环内只有一个语句的，也要另起一行
（4）循环内是空语句的，分号另起一行
实例:
// 1. 大括号不缩进（while/for 多语句块）
while (....)
{
    语句1;
    ...
    语句n;
}

for (....) {
    语句1;
    ...
    语句n;
}

// 2. 大括号缩进（for 多语句块）
for (....) {
    语句1;
    ...
    语句n;
}

// 3. 仅有一句语句时，不加大括号
while (....)
    语句;

for (....)
    语句;

// 4. 空语句（分号单独占行）
while (....)
    ;
for (....)
    ;

反例:
// 1. 单语句直接跟在 while/for 后，不换行
while(....) 语句;
for (....) 语句;

// 2. 空语句直接跟在 while/for 后，不换行
while (....);
for (....);

8、含有循环结构的程序(do-while）
要求：
（1）右括号不缩进，后面直接跟while
（2）左括号两种方式（直接在do后面/下一行）均可
（3）只有一个语句也加括号
实例:
// 写法1：左括号单独占行
do
{
    语句1;
    ...
    语句n;
} while(...);

// 写法2：左括号跟在do后，右括号不缩进
do {
    语句1;
    ...
    语句n;
} while(...);

// 写法3：紧凑格式，右括号直接跟while
do {
    语句1;
} while(...);

12、其它未列举的、更复杂的嵌套组合
以4个空格为单位，按层次依次缩进，每层比上层缩进四格
不符合要求者，每题扣10%的格式分
以本文档为准，后续所有编程作业均需要符合本文档的格式要求，后续不再有新的格式要求

严重违规:
1.打表输出
举例1：迭代法求2的平方根，直接cout〈く1.414;
举例2：1-9组成1953，直接540行输出

以下是快速理解:
### 1、正常顺序结构程序
**快速了解**：核心是代码按执行顺序逐行书写，所有语句统一缩进4格，一行仅写一条语句，左右括号单独成行，长语句可分行但保持对齐，通过注释和空行提升可读性，是最基础的代码排版规范。

### 2、含有单分支结构的程序
**快速了解**：if分支内语句缩进4格，左大括号可紧跟if或另起一行（二选一），右大括号单独成行；分支内仅一条语句时，可省略大括号但必须将语句另起一行，禁止语句与if同行。

### 3、含有双分支结构的程序
**快速了解**：if-else双分支均需遵循4格缩进，大括号规则同单分支；单语句分支省略大括号但需换行，多语句分支保留大括号且语句分行书写，禁止分支语句与if/else同行、多语句挤在同一行。

### 4、含有多分支结构的程序
**快速了解**：if-elseif-else链式结构为推荐写法，每层分支语句缩进4格；大括号、单/多语句排版规则与单/双分支一致，嵌套if-else虽不扣分但不推荐，核心是保证分支层级清晰、可读性强。

### 5、含有分支相互嵌套的程序
**快速了解**：嵌套的if分支按“每层缩进4格”原则逐层缩进，左大括号写法二选一，右大括号单独成行且建议缩进对齐；禁止左大括号另起一行+右大括号不缩进、右大括号层级错位等破坏可读性的写法。

### 6、含有switch/case的语句
**快速了解**：case比switch缩进4格且单独成行（case后不跟语句），case下的语句再缩进4格；switch的大括号排版需保证层级清晰，即使case下仅一条语句，也必须单独缩进一行，禁止与case同行。

### 7、含有循环结构的程序（while和for）
**快速了解**：循环内语句缩进4格，右大括号单独成行，左大括号写法二选一；单语句循环需换行（禁止与while/for同行），空语句循环的分号需单独占行，核心是区分循环关键字与循环体的层级。

### 8、含有循环结构的程序(do-while）
**快速了解**：do的左大括号可紧跟do或另起一行，循环体缩进4格；右大括号不缩进且直接紧跟while(...)，即使循环体仅一条语句，也必须保留大括号，是do-while与while/for排版的核心区别。

### 9、含有循环相互嵌套的程序
**快速了解**：嵌套的while/for循环按“每层缩进4格”逐层排版，左大括号写法二选一，右大括号单独成行；支持右大括号与对应循环关键字对齐，或随代码块缩进，核心是保证嵌套层级一目了然。

### 10、多种语句相互嵌套（基础组合）
**快速了解**：if、switch等语句嵌套时，按嵌套层次依次缩进4格，大括号排版遵循各自结构的规则，以“层级清晰、逻辑可区分”为核心，比如switch嵌套在if内时，switch整体缩进4格，case再相对switch缩进4格。

### 11、多种语句相互嵌套（循环嵌套）
**快速了解**：while/for循环相互嵌套时，内层循环整体比外层缩进4格，右大括号可与对应循环关键字对齐，或随代码块缩进；所有嵌套层级均以4个空格为单位，禁止层级错位、语句同行等问题。

### 12、其它未列举的、更复杂的嵌套组合
**快速了解**：无明确示例的复杂嵌套，统一遵循“4个空格为缩进单位，按层次依次缩进”的核心规则，每层代码比上层缩进4格，不符合则扣除对应格式分，是所有嵌套结构的通用兜底规范。

### 严重违规（通用提醒）
**快速了解**：禁止“打表输出”类投机写法（如直接输出计算结果、逐行硬编码输出内容），此类违规会直接判定格式不合格，核心要求是通过逻辑代码实现功能，而非直接输出结果。
